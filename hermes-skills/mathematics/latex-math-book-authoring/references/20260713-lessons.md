# 2026-07-13 教材写作重大教训

## 错误类型

| # | 错误 | 影响 | 修复 |
|---|------|------|------|
| 1 | ε-N/ε-δ 证明缺失（第3-5编空有 \lim 无 ε 定义） | 全书 30 章一度不可交付 | 逐章补充 16 个关键定理的 ε-δ 证明 |
| 2 | `begin{examplebox}[...}` 花括号错（写成 `}` 而非 `]`） | 编译报错 | 批量替换 + 自检清单 |
| 3 | `\blacksquare` 在数学模式外 | Missing $ inserted | 改为 `\(\blacksquare\)` |
| 4 | 连续 `\begin{exercise}` 缺 `\end{exercise}` | 环境不配对 | 自动修复 + 审计 |
| 5 | 习题忘记写（ch5,7,8,10,11,12 都漏了） | 编译成功但习题区空 | 写完正文后立即补，不可转头 |
| 6 | `^`/`_`/`\frac` 在 examplebox 标题中 | 花括号嵌套 | 简化标题或用 `{...}` 包裹 |
| 7 | `\end{section}` 不存在 | Undefined control sequence | 直接删除 |
| 8 | `\verb` 在 tabular/symbolcard 中 | 编译不稳定 | 用 `\texttt` 替代 |
| 9 | `openright` 空白页 | PDF 出现空白页 | 改为 `openany` |
| 10 | Unicode 字符 `→✓✗【】℃` 在 LaTeX 中 | Missing character | 全部替换为 LaTeX 命令或纯文本 |

## 根因

1. **赶进度**——30 章一次写完，没有逐章检查
2. **没有 ε 审计清单**——写完才意识到缺证明
3. **没有编译前自检**——写完直接编译撞墙
4. **没有修复层**——修复全靠手写，容易引入新 bug

## 解决方案

1. `epsilon-n-delta-audit` skill — ε 审计清单（已创建）
2. `mathbook-pipeline.py` — 统一管线，含 fix 层（已创建）
3. `latex-math-book-authoring` skill — 已嵌入 ε 审计入口
4. Memory — 已写入审计铁律

## 核心教训

**写教材的流程应该是：**
```
创建骨架 → 写内容 → 补习题 → 审计（ε 追踪）→ 修复（分层）→ 编译 → 报告
```
**任何时候都不可以跳过审计和修复直接编译。**
