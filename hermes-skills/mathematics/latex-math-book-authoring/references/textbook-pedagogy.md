# 中文数学教材编写守则（系统提示词摘录）

源自《自学高数》教材生成系统提示词。每项规则可独立应用于任何 LaTeX 数学教材章节。

## 1. 应用引导（每章开头必写）

回答三个问题：
1. 这章学的东西在真实世界解决什么具体问题？
2. 学完这章之前我做不到什么？学完之后我能做到什么？
3. 给一个具体的、日常的例子（不准用"物理学中……"，要用"你手机里的导航软件用这个算最短路径"这种级别的例子）

## 2. 先直觉，再严格

每个概念的展开顺序（不可调换）：
1. 直觉引入：一个故事、一个问题、一个生活中的场景
2. 可视化描述："想象你在爬一座山……"
3. 非严格定义：用自然语言说清楚
4. 严格定义（定义框）：放在直觉之后，不是之前
5. 立即跟一个例子：数字算一遍的那种

## 3. 每个公式都要"读出来"

所有数学符号必须标注：
- 符号长什么样（LaTeX 书写）
- 符号怎么读（汉字注音，如"∑ 读作西格玛"）
- 用一句话说清楚"这个符号在干什么"

在符号首次出现处使用 `symbolcard` 环境，不塞进附录。

## 4. 练习与答案规范

- 练习题编为 `练习 n.m`（n=章号, m=题号）
- 每道题标注难度：[易]、[中]、[难]
- 答案放在附录 A，简单题只给答案，中档题给关键步骤+答案，难题给完整推导

## 5. 禁忌词（不要在教材正文中出现）

| 不要写 | 要写 |
|--------|------|
| "显然" | 因为……所以…… |
| "不难看出" | 我们来看实际数据…… |
| "易证" | 我们一步一步验证…… |
| "由定义可知" | 回顾一下定义…… |
| "众所周知" | （直接说事实，不加前缀） |
| "考虑"（后面跟空洞推理） | 看这个具体例子…… |

## 6. 数值验证（每个公式结论都验证）

教材中每个数学结论（积分结果、微分方程解、极限值、导数表达式）必须用 Python 数值验证。验证标准：
- 积分/导数：解析结果 vs 数值近似，相对误差 < 1e-8
- 极限：代入越来越接近的数值看趋势
- 微分方程的解：代回原方程检查

验证代码放入 `lstlisting` 块（注意：代码块中 `_`、`<`、`>` 等字符需要用 `\verb` 或 `lstlisting` 模式处理）。

## 7. 颜色体系

| 元素 | 颜色 | tcolorbox 样式 |
|------|------|----------------|
| 定义（定义框） | 墨绿色 | `colframe=green!60!black` |
| 定理（定理框） | 深蓝色 | `colframe=blue!60!black` |
| 命题（命题框） | 蓝紫色 | `colframe=purple!60!black` |
| 例题（例题框） | 暖橙色 | `colframe=orange!70!black` |
| 注意（注意框） | 暗红色 | `colframe=red!70!black` |
| 应用引导 | 青绿色 | `colframe=teal!60!black` |
| 进步宣言 | 金黄 | `colframe=yellow!60!orange` |
| 符号卡片 | 深灰 | `colframe=gray!50!black` |
