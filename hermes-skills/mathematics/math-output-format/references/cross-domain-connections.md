# 跨领域连接模式（Cross-Domain Connections）

## 何时启用

用户提出了一个问题，而你发现可以连接到一个数学结果和一个真实世界数据，且两者数字精确吻合或形成有趣对照。这不是"讲个趣闻"——是用真实世界的数据为抽象数学做锚定。

## 典型信号

- 用户提到一个数学常数（1/e, π, φ, e 等），它"恰好"出现在另一个完全不相关的场景
- 问题同时涉及"数学原理"和"社会/经济/物理现象"
- 你意识到自己可以说出"两个 X 的对话"

## 模式模板

### 1. 数学端推导 + 验证
```bash
# SymPy 验证最优性
source ~/.hermes/profiles/xiandaishuxuejia/.venv/bin/activate
python3 -c "
import sympy as sp
x = sp.symbols('x', positive=True)
P = x * sp.log(1/x)
dP = sp.diff(P, x)
sp.solve(sp.Eq(dP, 0), x)  # → [exp(-1)]
"

# 数值模拟验证
python3 -c "
import numpy as np
# 模拟秘书问题，验证 1/e ≈ 37% 确实是最优斩杀比例
# ... (见具体 session 代码)
"
```

### 2. 真实世界数据获取
直接从权威来源获取最新数据，而非依赖记忆中的旧数字：
```
浏览器 → 官方报告 → 提取关键百分比
```
对比数学结果：`|数学比例 - 真实比例| < 几个百分点` → 成立

### 3. 双栏对照表
将数学和现实并排呈现，让用户一目了然看到对应关系：

```
| | 数学世界 | 现实世界 |
|---|---|---|
| 数值 |  1/e ≈ 36.79% | 37% |
| 含义 | 秘书问题最优斩杀比例 | 真实场景的对应解读 |
| 来源 | 最优停止理论 | 美联储 SHED 报告 |
```

### 4. 点睛收尾（可选，有 wit 时加分）
数学结论和现实结论之间形成一个精炼的总结句，让用户觉得"这个连接我没想到但很自然"。

## 注意事项

- ❌ **不要强行连接**——如果数字只是近似（如 36% vs 37%）可以说"接近"，但相差超过 3 个百分点就不要用"恰恰是"的语气
- ✅ **两端都要独立验证**——数学用 SymPy，现实数据从权威来源（Fed、BLS、WHO 等）的官网实时获取，**不要依赖 AI 记忆**
- ✅ **用 wit 但不要过度**——用户喜欢的是"这个联系很妙"，不是"看我多聪明"
- ✅ **保持严肃性**——跨领域连接≠搞笑梗，核心是数学和现实的精确对应
